# CLI 重写与升级计划

## 一、CLI 自身 Node 依赖升级（`packages/cli/package.json`）

当前版本 → 升级目标：

| 包 | 旧版本 | 新版本 | 备注 |
|---|---|---|---|
| chalk | ^4.1.2 | ^5.3.0 | ESM-only，需配合 ESM 迁移 |
| commander | ^12.0.0 | ^13.1.0 | |
| inquirer | ^8.2.6 | ^12.5.0 | ESM-only，v9+ API 有破坏性变更，prompt 改为单独 import |
| fs-extra | ^11.2.0 | ^11.3.0 | |
| ora | ^5.4.1 | ^8.2.0 | ESM-only；需在所有 inquirer prompt 结束后才调用 `.start()`，避免 stdout 冲突 |
| ejs | ^3.1.9 | ^3.1.10 | |
| update-notifier | 新增 | ^7.3.1 | 启动时异步检查 npm 新版本并提示用户升级 |
| node engines | >=18.12.0 | >=20.0.0 | |

同时新增 `"type": "module"` 到 `package.json`，并将版本升级为 `2.0.0`。

## 二、CLI 代码重构为 ESM（全部文件）

### 2.1 `bin/artisan.js`
- `require()` → `import`
- 新增 `--js / --ts` flag 供用户选择生成模板类型
- 新增 `info` 命令展示版本/依赖信息
- 集成 `update-notifier`，启动时异步检查新版本

### 2.2 `lib/create.js`
- 全面改写为 ESM，使用 inquirer v12 新 API（`import { input, confirm, select } from '@inquirer/prompts'` 形式）
- `createMainApp` / `createSubApp` 新增 `language`（js|ts）询问，决定使用哪套模板
- 确保所有 prompt 结束后再启动 ora spinner

### 2.3 `lib/generator.js`
- 全面改写为 ESM
- 支持按 `templateName` 在 `templates/<type>-js/` 或 `templates/<type>-ts/` 目录选择

### 2.4 `lib/utils.js`
- 全面改写为 ESM

### 2.5 `lib/index.js`（新建）
- 修复现有 bug：`package.json` 中 `"main": "lib/index.js"` 但该文件不存在
- 作为程序化 API 入口，导出 `createMainApp` / `createSubApp` / `generateProject`

## 三、模板结构扩展（`packages/cli/templates/`）

每种应用类型提供两套模板：`xxx-js/` 和 `xxx-ts/`，模板文件内容直接从当前 `packages/` 下对应子应用复制，将硬编码的项目名/端口替换为 EJS 变量。

> **注意**：执行时先建好所有新模板目录并验证生成可用，最后再删旧目录（避免过渡期报错）。

## 四、模板文件中的 EJS 变量约定

| 变量 | 说明 | 使用位置 |
|---|---|---|
| `<%= name %>` | 项目名称（npm package name） | 所有 package.json.ejs |
| `<%= description %>` | 项目描述 | 所有 package.json.ejs |
| `<%= port %>` | 开发服务器端口 | vite.config.js.ejs / vue.config.js.ejs / publicPath |
| `<%= appName %>` | qiankun 注册名（子应用唯一标识） | vue3/vue2 sub-app vite.config / vue.config |

## 五、执行顺序

> 核心原则：**先建模板 → 再改逻辑 → 最后删旧目录**，确保每步可单独验证。

1. 更新 `packages/cli/package.json`（ESM + 升级依赖 + 新增 update-notifier）
2. 新建 `lib/index.js`（修复 main 字段 bug，导出公共 API）
3. 重写 `lib/utils.js` → ESM
4. 重写 `lib/generator.js` → ESM（支持 js/ts 双模板目录）
5. 重写 `lib/create.js` → ESM（新增 language 询问，使用 inquirer v12 新 API）
6. 重写 `bin/artisan.js` → ESM（新增 --js/--ts flag + info 命令 + update-notifier）
7. 创建 `templates/main-app-js/` 完整模板（从 packages/main-app 提取，替换 EJS 变量）
8. 创建 `templates/main-app-ts/` 完整模板（TS 变体，含 tsconfig/env.d.ts/ts 后缀源文件）
9. 创建 `templates/vue3-sub-app-js/` 完整模板
10. 创建 `templates/vue3-sub-app-ts/` 完整模板
11. 创建 `templates/vue2-sub-app-js/` 完整模板（补充 babel.config.js）
12. 创建 `templates/iframe-sub-app-js/` 完整模板
13. 删除旧模板目录（`templates/main-app/`、`templates/vue3-sub-app/`、`templates/vue2-sub-app/`、`templates/iframe-sub-app/`）
