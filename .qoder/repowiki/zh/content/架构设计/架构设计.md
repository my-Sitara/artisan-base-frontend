## 简介
Artisan 微前端平台是一个企业级的 Monorepo 脚手架，采用 Lerna + npm workspaces 管理多包结构，主应用基于 Vue3 + Vite + qiankun（loadMicroApp 模式），支持多种类型的子应用（Vue3/Vue2/iframe/link）。平台提供完整的跨应用通信桥（bridge）、**新增布局编排系统**、状态持久化（Pinia + 持久化插件）、iframe 跨域治理与安全策略，并支持同屏多应用实例加载。

**更新** 新增了完整的布局管理系统，支持四种布局类型（default、full、embedded、blank）和智能约束机制，提供灵活的页面布局控制能力。

## 项目结构
仓库采用 Monorepo 架构，根目录通过 Lerna 和 npm workspaces 统一管理各子包。核心目录与职责如下：
- packages/main-app：主应用，负责微应用编排、状态管理、跨应用通信桥初始化、容器组件与 qiankun 集成、**新增布局管理器**
- packages/vue3-sub-app：Vue3 子应用示例，使用 vite-plugin-qiankun 提供生命周期与沙箱隔离
- packages/vue2-sub-app：Vue2 子应用示例，兼容 qiankun 生命周期与路由抽象模式
- packages/iframe-sub-app：iframe 子应用示例，演示 iframe 类型微应用的接入与治理
- packages/cli：脚手架 CLI，用于快速生成主应用与各类子应用模板
- user-docs：**新增VitePress文档站点**，包含API参考和布局系统指南

## 核心组件
- 微应用管理器（MicroAppManager）：封装 qiankun 的 loadMicroApp，统一管理应用生命周期、预加载、心跳检测、错误日志与样式快照等
- **新增布局管理器（LayoutManager）**：提供四种布局类型管理、布局选项配置、约束验证和变化监听
- 微应用容器组件（MicroAppContainer）：提供加载遮罩、错误展示、重试机制与容器生命周期绑定
- 主应用入口（main.js）：初始化 Pinia/Pinia持久化、Element Plus、路由与桥接（bridge）；暴露微应用管理器便于调试
- Vue3 子应用入口（main.js）：通过 vite-plugin-qiankun 注册生命周期；qiankun 模式下使用 memory history 并避免直接 unmount
- Vue2 子应用入口（main.js）：兼容 qiankun 生命周期；qiankun 模式下使用 abstract 路由模式
- **新增布局配置模块（layoutConfig）**：定义布局类型约束、默认选项和配置标准化机制
- **新增布局API模块（layoutAPI）**：支持mock和API两种模式的布局数据持久化
- CLI 工具：提供创建主应用与子应用的命令行能力

## 架构总览
整体架构围绕"主应用 + 多子应用"的微前端模式展开，主应用通过 qiankun 的 loadMicroApp 动态加载子应用，子应用在独立运行或被主应用装载时分别采用不同的路由模式与资源加载策略。跨应用通信通过桥接（bridge）实现，状态管理采用 Pinia 并结合持久化插件。**新增的布局系统提供灵活的页面布局控制能力**。

## 详细组件分析

### 微应用管理器（MicroAppManager）
- 职责
  - 统一加载/卸载微应用，支持 Vue3/Vue2/iframe/link 类型
  - 预加载策略与状态缓存（preloaded、lastModified）
  - 心跳检测与错误日志记录
  - 样式快照与沙箱隔离（experimentalStyleIsolation）
  - 令牌透传与主应用路由注入
- 关键流程
  - load(appId, container, options)：校验配置与状态，区分 iframe/link 与 qiankun 加载路径，设置 sandbox 与 props
  - unload(appId)：清理内存引用与容器内容
  - isAppLoaded(appId)：防重复加载
  - loadIframe/loadLink：iframe 直接注入容器；link 打开新窗口
- 性能与稳定性
  - 使用 markRaw 降低响应式开销
  - 通过容器元素而非选择器字符串传入，避免 DOM 变更导致的定位失效
  - 对样式进行快照与隔离，减少全局污染

## API参考体系

### MicroAppManager API
- **导入方式**：`import { microAppManager } from '@/core/microAppManager'`
- **核心方法**
  - `load(appId, container, options)`：加载微应用
  - `unload(appId)`：卸载微应用
  - `reload(appId)`：刷新微应用
  - `preload(appIds)`：预加载微应用资源
- **状态管理**
  - `setAppStatus(appId, status)`：设置应用状态
  - `isAppLoaded(appId)`：检查应用是否已加载
  - `getLoadedCount()`：获取已加载应用数量
- **错误管理**
  - `getErrorLogs(appId?)`：获取错误日志
  - `clearErrorLogs()`：清空错误日志
- **属性**
  - `loadedApps`：已加载应用信息
  - `errorLogs`：错误日志数组
  - `preloadStatus`：预加载状态映射

### Bridge API
- **导入方式**：`import { bridge } from '@/core/bridge'`
- **消息处理**
  - `on(type, handler)`：注册消息处理器
  - `off(type)`：移除消息处理器
  - `send(targetWindow, message, targetOrigin?)`：发送消息到指定窗口
  - `broadcast(message)`：广播消息给所有子应用
- **Token管理**
  - `syncToken(token)`：同步token到所有子应用
- **导航功能**
  - `navigateTo(options)`：跳转到子应用
  - `navigateToMain(path, query?)`：跳转到主应用
- **生命周期**
  - `destroy()`：销毁bridge，移除监听器

### LayoutManager API
- **导入方式**：`import { layoutManager } from '@/core/layoutManager'`
- **布局设置**
  - `setLayout(type, options?)`：设置布局类型和选项
  - `updateOptions(options)`：更新布局选项
  - `setLayoutFromMicroApp(microAppConfig)`：根据微应用配置设置布局
  - `reset()`：重置为默认布局
- **获取信息**
  - `getLayoutType()`：获取当前布局类型
  - `getLayoutOptions()`：获取当前布局选项
- **监听变化**
  - `onChange(callback)`：监听布局变化
  - `offChange(callback)`：移除监听器

### 配置API
- **微应用配置结构**
  - `id`：唯一标识（必须与 vite-plugin-qiankun 注册名一致）
  - `name`：应用名称
  - `entry`：入口地址（完整 URL）
  - `activeRule`：激活规则（路由前缀）
  - `type`：应用类型（'vue3' | 'vue2' | 'iframe' | 'link'）
  - `layoutType`：布局类型（'default' | 'full' | 'embedded' | 'blank'）
  - `layoutOptions`：布局选项（包含 showHeader、showSidebar、showFooter、keepAlive）
  - `status`：状态（'online' | 'offline'）
  - `version`：版本号
  - `lastModified`：最后修改时间戳
  - `preload`：是否预加载
  - `props`：传递给子应用的属性
- **配置标准化**
  - 使用 `normalizeLayoutConfig()` 自动应用布局约束
  - 配置中与约束冲突的选项会被覆盖
- **环境变量**
  - `VITE_USE_MICRO_APPS_API`：true/false，控制从API加载还是使用mock

## 故障排查指南
- 加载失败
  - 检查容器是否存在与可访问
  - 查看错误事件与 errorLogs，确认网络/入口/状态
- 重复加载
  - 确认 isAppLoaded 判定逻辑，避免同一 appId 多次加载
- 卸载异常
  - Vue3：不要直接调用 app.unmount，改用停止响应式作用域与清理引用
  - Vue2：确保 $destroy 正常执行并清理容器内容
- 路由不生效
  - qiankun 模式下需传入 subPath 并在 mount 后手动 push 初始路由
- 跨域与样式污染
  - 开启沙箱隔离与样式快照；iframe 场景遵循跨域治理策略
- **新增** 布局相关问题
  - 布局切换不生效：检查布局类型是否在支持范围内，确认约束规则
  - 布局配置无效：确认配置是否符合布局类型的约束条件
  - 布局API异常：检查环境变量配置，确认API模式下的降级机制
  - 布局持久化失败：检查localStorage权限和网络连接状态
