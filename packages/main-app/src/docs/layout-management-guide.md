# 布局管理功能说明

## 功能概述

布局管理功能允许在应用管理页面中动态调整子应用的布局配置，并在子应用加载时应用相应的布局。当在应用管理页面中修改子应用的布局配置时，系统会在子应用下次加载时应用新的布局。

## 核心组件

1. **LayoutManager** - 全局布局管理器，负责管理当前的布局状态
2. **App.vue** - 主应用入口，根据微应用配置动态选择布局组件
3. **SubAppPage.vue** - 子应用页面，加载时应用对应微应用的布局配置

## 使用方法

### 在应用管理页面

当用户在应用管理页面修改子应用的布局配置并保存时：

1. 系统调用 `appStore.updateApp()` 更新微应用配置
2. 当用户访问子应用时，主应用会根据微应用配置中的布局类型动态选择布局组件

### 在主应用中

App.vue 会根据当前路由和微应用配置动态选择布局：

1. 检测当前路由是否为子应用路由（通过 appId 或 params.appId）
2. 根据 appId 获取微应用配置
3. 使用微应用配置中的 layoutType 和 layoutOptions
4. 应用相应的布局组件

### 在子应用页面

SubAppPage.vue 会在加载时应用对应微应用的布局配置：

1. 根据当前 appId 获取微应用配置
2. 使用 layoutManager.setLayout() 应用微应用的布局配置

## 配置选项

- **layoutType**: 布局类型 (default, tabs, full, embedded, blank)
- **layoutOptions**: 布局选项
  - showHeader: 是否显示头部
  - showSidebar: 是否显示侧边栏
  - keepAlive: 是否启用KeepAlive
  

## 实现细节

布局管理功能通过动态路由和微应用配置结合实现。当布局配置发生改变时，系统会在子应用下次加载时应用新的布局，提供了灵活的布局管理能力。
